# 降低车轮速度参数

## 需求

电机速度接口已统一为 `-100~100` 百分比后，车轮表现异常且无法稳定循迹。排查发现不是单个 PID 参数问题，而是控制链路存在多个量纲/状态问题：APP 曾把编码器单次采样 `delta` 当作控制周期速度反馈；行车模式被调试代码绕过；控制周期为 1ms 时目标速度与软件轮询编码器能力不匹配；循迹外环差速输出过大；循迹方向符号反了；循迹丢线时 reset 过重。

## 验收标准

- 循迹 APP 和直行 APP 的速度反馈均使用编码器累计 `count` 差值，单位恢复为 `counts/control-period`。
- 编码器驱动层 `delta` 继续表示单次采样增量，不直接作为 APP 速度环反馈。
- 基础目标速度和速度环输出限幅进入低速调试档，避免仍然接近满占空比。
- 行车模式恢复为 Key1 启动直行、检测灰线后切入循迹。
- 控制周期调整为 20ms，速度目标解释为每 20ms 的编码器计数差。
- 循迹/直行外环差速限幅收敛，避免外环直接生成过大的左右轮目标速度。
- 循迹修正转向符号：线在右侧时左轮快、右轮慢。
- 循迹丢线时只停止电机，不重置 PID 和编码器基准，避免短暂丢线导致状态反复清空。
- 工程门禁 `ARCH/HW/SPEC` 通过。

## 约束

- 电机 DAL 接口继续使用 `-100~100` 百分比。
- BSP PWM 内部继续使用千分比占空比。
- 本次修正控制链路量纲、模式状态机和低速调试参数，不做硬件编码器中断化改造。

## MVP 边界

- 循迹 APP 和直行 APP 恢复必要的编码器上一周期累计计数缓存，用 `count - last_count` 计算速度反馈。
- 循迹基础速度使用 `8 counts/control-period`，直行基础速度使用 `6 counts/control-period`。
- 左右速度环使用 `Kp=8.0, Ki=0.05, output_limit=60, iterm_limit=20`。
- APP 控制周期统一为 `20ms`。
- 循迹外环输出限幅为 `4 counts/control-period`，直行角度外环输出限幅为 `4 counts/control-period`。

## 已排除项

- 不调整灰度权重。
- 不调整编码器驱动。
- 不调整直行/循迹基础目标速度。
