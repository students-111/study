# 编码器GPIO双边沿中断解码改造

## 需求 / 验收标准
- M1、M2 两路正交编码器同时从 1 ms GPIO 轮询改为 GPIOB 双边沿中断解码。
- 四个 A/B 相引脚均按上升沿和下降沿触发，保持现有四相转移表的四倍频计数语义。
- 保持 `speed_cp` 为 10 ms 测速周期内的累计计数差，不改变 APP/PID 使用单位。
- 固定 PWM 提升时，`speed_cp` 不再受轮询方案约 10 counts/10 ms 的软件上限限制。
- M1、M2 正转时速度符号保持为正，安装方向修正保持现有配置。
- 构建、SysConfig 生成检查、架构门禁和硬件资源锁检查全部通过。

## 约束
- 芯片为 MSPM0G3507；M1 使用 PB15/PB16，M2 使用 PB12/PB13，四路均属于 GPIOB/Group1。
- 固定引脚、中断边沿和中断使能优先写入 `empty.syscfg`，不得手工编辑生成文件。
- ISR 只读取 A/B 相、执行正交解码、更新计数和快照，不打印、不执行 PID、不阻塞。
- 生产代码依赖方向保持 APP -> DAL -> Driver/BSP -> DriverLib/SysConfig。
- APP 继续只读 `g_dal_encoder_sample[]`，不得新增 latest/getter 拷贝接口。
- 保留当前人话注释，新增或修改的注释使用简体中文。

## MVP 边界
- 同时支持 M1、M2 两路四倍频双边沿中断解码。
- 停用 1 ms 编码器轮询任务，保留 10 ms 速度差分任务。
- 为 ISR/主循环共享计数与快照提供 `volatile` 和必要的短临界区保护。

## 已排除项
- 不修改循迹、直行或 mode APP 策略。
- 不调整速度 PID、循迹 PID、电机 PWM 参数或编码器机械换算系数。
- 不新增 RPM、m/s 等物理单位换算。
- 不在本轮加入滤波、速度预测或编码器故障诊断状态机。
