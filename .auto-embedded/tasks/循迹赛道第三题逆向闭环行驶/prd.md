# 循迹赛道第三题逆向闭环行驶

## 需求 / 验收标准
- 小车在 A 点就位、车头正对 B 点，按 Key1 后按 A→C→B→D→A 的顺序自动完成一圈，并在 A 点停车。
- A→C、B→D 为白底直线段：先原地转到目标航向，再使用现有直线保持能力行驶；重新检测到黑线即抵达 C、D。
- C→B、D→A 为黑色半圆弧：C、D 停车 1 s 后，若灰度仍检测到弧线则直接进入循迹；若未检测到黑线，再按指定方向原地转至捕获弧线后进入循迹；黑线消失即抵达 B、A。
- C、B、D 节点均停车 1 s，再执行下一段的原地转向；到达 A 后保持停车。
- A→C、B→D 的原地转向以 MPU6050 相对 Yaw 为闭环判据；C→B、D→A 以“先离线、再捕线”的灰度判据进入循迹。

## 约束
- 赛道仅有两段黑色半圆弧，其余区域为白底；节点无额外定位标记。
- 黑线出现或消失须连续 3 个灰度刷新周期一致才确认，避免边缘抖动误切换。
- 正常到点、节点暂停和最终停车时，左右轮速度 PID 目标设为 0；不以直接失能电机作为正常停车方式。
- 复用既有 `app_line_follow`、`app_straight_drive` 与 `dal_motor`；第三题状态机仍位于 `app.c`，不使用 `app_drive_mode`。
- `app.c` 同时保留 task2 与 task3 两套状态机；当前构建选择 task3，且 task3 沿用 task2 的 Key1 启动方式。
- Yaw 正负方向及原地转向的实际角度必须上板校准，不能仅凭题图符号直接固化。

## 初始角度预测
- 以 A 点车头朝 B 的方向为 0°：A→C 约 -50.2°，B→D 约 +38.7°。
- 上述符号仅为几何坐标推导；应以 MPU6050 的实际 Yaw 增长方向为准调整。

## MVP 边界
- 包含：Key1 启动、A/B 两次相对角原地转向、C/D 两次离线再捕线转向、两段白底直线、两段弧线循迹、C/B/D 三次 1 s 停车、A 点最终停车。
- 不包含：声光提示、自动角度标定、赛道脱线后的自动重定位、速度优化。
