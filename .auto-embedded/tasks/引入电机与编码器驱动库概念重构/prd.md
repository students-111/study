# 引入电机与编码器驱动库概念重构

## 需求 / 验收标准
- 新增电机驱动库概念，使用类似 `Motor_t` 的对象描述单个电机。
- `Motor_t` 需要包含硬件绑定字段、当前速度和当前方向等运行状态。
- 电机库不直接绑定具体 APP 业务语义，硬件操作直接调用 BSP 抽象接口完成。
- 现有左右电机控制需要逐步改为基于电机驱动库对象封装。
- 新增编码器驱动库概念，风格与电机驱动库一致，使用简单对象实体维护硬件配置、计数、增量、状态、使能和安装方向。
- 新增 `User/Driver/` 层：`drv_motor.*` 和 `drv_encoder.*` 作为可复用驱动库；`DAL` 负责创建左/右电机、M1/M2 编码器实例并注册 BSP 回调。
- MVP 要求 APP 调用面保持不变，仍通过 `dal_motor_*`、`g_dal_encoder_sample[]` 使用电机和编码器能力。

## 约束
- 需求处于 brainstorm 阶段，先收敛 MVP 和分层边界，再进入 PLAN。
- 保持 APP -> DAL -> BSP/DriverLib 分层方向，不让 APP 直接依赖底层硬件细节。
- 不修改 SysConfig 固定硬件资源，除非后续确认需要新增外设资源。
- Driver 层不得包含 `ti_msp_dl_config.h` 或直接调用 DriverLib；电机 Driver 可以直接调用 BSP 抽象接口，避免 DAL 内部堆积转发函数。
