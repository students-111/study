# 驱动库移植完整流程参考

> 本文件由主协议按需加载。INNOVATE/EXECUTE 阶段需要移植外设驱动时读取此文件。

---

## 搜索优先级（每步指定使用的工具）

> **英飞凌芯片特殊规则**：如果当前芯片为英飞凌系列（TC264、TC377 等），**优先触发逐飞库管理模式**（`.auto-embedded/modes/seekfree-lib.md`），从 https://gitee.com/seekfree 获取成熟驱动，而非按以下通用流程搜索。

1. **官方库文档**：用 **Context7 MCP** 查询目标固件库（HAL / ESP-IDF / Arduino / StdPeriph）的 API 文档与使用示例，确认标准用法
2. **GitHub/Gitee 开源驱动**：用 `gh` CLI 搜索 GitHub 仓库，或用 grok-search 搜索 Gitee 仓库，关键词 `芯片型号 + 外设名 + driver/lib`
3. **论坛/竞赛口碑驱动**：用 grok-search 搜索 CSDN、嘉立创开源广场、立创 EDA 社区中的评价和使用记录
4. **数据手册/时序图**：用 `/playwright-skill` 打开芯片/模块官网或 PDF 数据手册页面，提取关键时序参数

---

## 移植评估标准

拿到候选驱动后，按以下标准快速评估，**全部通过才移植**：

| 评估项 | 要求 |
|--------|------|
| 接口风格 | 函数前缀清晰、有初始化/读/写分离 |
| 依赖范围 | 仅依赖目标固件库（StdPeriph/HAL/ESP-IDF），无跨平台污染 |
| 资源占用 | Flash/RAM 估算在预算内 |
| 时序正确性 | 有数据手册对应或注释说明关键时序来源 |
| 可读性 | 代码可读，无大量魔术数字，理解后可维护 |

---

## 移植步骤

1. **适配层替换**：将原驱动的底层读写函数（如 HAL_SPI_Transmit）替换为项目固件库的等价调用
2. **命名规范化**：按主协议命名规范调整函数前缀和文件名
3. **去冗余**：删除项目用不到的功能（降低 Flash 占用）
4. **注明来源**：在文件头注释中记录驱动来源 URL 和移植日期

---

## 回退条件（以下情况才自行实现）

- 搜索后确实无合适驱动存在
- 现有驱动依赖与项目固件库不兼容且适配成本过高
- 驱动代码质量极差（无法理解、时序未说明）
- 项目对 Flash/RAM 有极严格限制，现有驱动过于臃肿
