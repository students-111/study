> **分工**：本文是**症状速查表**（一眼看症状 → 知道往哪查）。复杂或未知根因问题走 **`.auto-embedded/refs/systematic-debugging.md`**（四阶段根因分析方法论）。两文件互补，不要互相展开内容。

## 按症状诊断

### 通信类故障

| 症状 | 可能原因 | 快速验证 | 解决方案 |
|------|---------|---------|---------|
| 串口乱码 | 波特率不匹配 / 数据位/停止位配置错误 | 串口回环测试（TX 接 RX） | 检查双方波特率、数据位（8）、停止位（1）、校验位（None） |
| 串口无数据 | 引脚配置错误 / 外设时钟未使能 | 示波器测量引脚波形 | 检查 GPIO 复用配置、RCC_APBxENR 时钟使能 |
| I2C ACK 失败 | 从机地址错误 / 上拉电阻问题 | 用逻辑分析仪抓取 SDA/SCL | 检查从机地址、SDA/SCL 上拉（4.7kΩ） |
| SPI 数据错误 | 时钟极性 / 相位配置错误 | 示波器测量 SCK/MOSI | 检查 SPI_CR1 中的 CPOL/CPHA 位 |

### 外设类故障

| 症状 | 可能原因 | 快速验证 | 解决方案 |
|------|---------|---------|---------|
| 定时器无输出 | 时钟未使能 / PWM 模式未配置 | 示波器测量引脚 | 检查 RCC_APB1ENR/RCC_APB2ENR、TIMx_CCMR 中的 PWM 模式位 |
| PWM 占空比错误 | 自动重装载值计算错误 | 串口打印 ARR/CCR 值 | 验证公式：占空比 = (CCR+1) / (ARR+1) |
| ADC 读数异常 | 采样时间不足 / 输入阻抗过大 | 并联已知电压源测试 | 增加采样周期（ADC_SMPR）、检查输入电路 |
| 外设无响应 | 引脚未复用 / 复用功能冲突 | 读取 GPIO_MODER 寄存器 | 检查 GPIO 模式配置（AF00/AF01 等） |

### 中断类故障

| 症状 | 可能原因 | 快速验证 | 解决方案 |
|------|---------|---------|---------|
| 中断不触发 | NVIC 未配置 / 中断标志未清除 | 在中断入口设置断点 | 检查 NVIC_EnableIRQ、中断清除标志位 |
| 中断频繁触发 | 中断源未屏蔽 / 中断优先级冲突 | 查看中断计数器 | 检查 EXTI_IMR、调整中断优先级 |
| 硬件中断丢失 | 中断处理时间过长 / 中断嵌套过深 | 测量中断处理时间 | 优化中断处理函数、降低中断优先级 |

### 启动类故障

| 症状 | 可能原因 | 快速验证 | 解决方案 |
|------|---------|---------|---------|
| 代码烧录后不运行 | 启动模式错误 / 向量表配置错误 | 检查 STRAP 引脚电平 | 查阅 datasheet 启动模式引脚（如 STM32 的 BOOT0/BOOT1） |
| 程序卡在启动位置 | 时钟配置错误 / 外部晶振问题 | 测量时钟引脚 | 检查 SystemInit 中的时钟树配置、晶振起振 |
| Flash 擦除失败 | Flash 时钟超频 / Flash 写保护 | 查看 Flash 状态寄存器 | 降低 Flash 时钟频率、解除 Flash 写保护 |

### 存储类故障

| 症状 | 可能原因 | 快速验证 | 解决方案 |
|------|---------|---------|---------|
| 全局变量初始化异常 | .data 段未正确链接 / RAM 不足 | 单步调试启动代码 | 检查链接脚本、增加 RAM 空间 |
| 堆栈溢出 | 递归过深 / 局部数组过大 | 查看栈指针值 | 增大栈空间、优化递归调用 |
| Flash 空间不足 | 代码过大 / 常量过多 | 查看编译后的 size 输出 | 优化代码、将大数组移至外部 RAM |

## 按芯片平台排查

### STM32 系列常见问题

**问题**：GPIO 不工作
**检查顺序**：
1. GPIO_MODER 是否设置为正确模式（输入/输出/复用）
2. RCC_AHBxENR 时钟是否使能
3. 是否需要 SYSCFG remap

**问题**：定时器不启动
**检查顺序**：
1. RCC_APBxENR 是否使能对应定时器时钟
2. TIMx_CR1 中的 CEN 位是否置 1
3. 中断是否使能（如使用中断模式）

### ESP32 系列常见问题

**问题**：串口无输出
**检查顺序**：
1. GPIO0/1 是否未用作其他功能
2. UART 模块时钟是否使能
3. 引脚矩阵（IO_MUX）是否正确配置

**问题**：WiFi 连接失败
**检查顺序**：
1. 天线是否连接
2. WiFi 模式（STA/AP）是否正确
3. WiFi 配置（ssid/password）是否正确

### Arduino 常见问题

**问题**：digitalWrite 无效
**检查顺序**：
1. 引脚号是否正确（如 UNO 的 0-13）
2. pinMode 是否已调用
3. 是否与 PWM/SPI/I2C 引脚冲突

## 快速诊断工具使用

### 使用示波器

| 检测项 | 测量方法 | 正常波形特征 |
|--------|---------|------------|
| 时钟信号 | 探头接时钟引脚 | 规则方波，频率与配置一致 |
| PWM 输出 | 探头接 PWM 引脚 | 方波，占空比与计算值一致 |
| 串口 TX | 探头接 TX 引脚 | 起始位（低）→ 数据位 → 停止位（高） |
| I2C SCL | 探头接 SCL 引脚 | 时钟方波，ACK 时有下拉 |

### 使用逻辑分析仪

**I2C 诊断**：
- START 信号：SDA 高→低，SCL 高
- STOP 信号：SDA 低→高，SCL 高
- ACK/NAK：在 SCL 第 9 个时钟周期检测 SDA 电平

**SPI 诊断**：
- CS 时序：在数据传输前/后正确拉低/拉高
- 时钟极性：空闲时为低（CPOL=0）或高（CPOL=1）
- 相位：在时钟边沿采样（CPHA=0/1）

### 使用串口调试

**回环测试步骤**：
1. TX 接 RX（用导线或跳线）
2. 发送测试数据
3. 检查接收数据是否一致

**常用测试命令**：
```c
// STM32 HAL 示例
HAL_UART_Transmit(&huart1, "TEST\r\n", 6, 100);

// ESP-IDF 示例
uart_write_bytes(uart_num, (const uint8_t *)"TEST\r\n", 6);
```

## 常见错误代码速查

### Keil 编译错误

| 错误代码 | 含义 | 解决方案 |
|---------|------|---------|
| L6205E | 符号未定义 | 检查拼写、包含头文件 |
| L6218E | 重复定义 | 检查头文件 include 保护 |
| L6316E | 无法打开源文件 | 检查文件路径、路径分隔符 |

### GCC 编译错误

| 错误类型 | 示例 | 解决方案 |
|---------|------|---------|
| undefined reference | `undefined reference to 'foo'` | 检查变量声明、链接对象文件 |
| multiple definition | `multiple definition of 'foo'` | 检查头文件 include 保护、避免 .c 中定义全局变量 |
| implicit declaration | `implicit declaration of function 'foo'` | 包含正确的头文件 |

### 运行时错误（Cortex-M）

| 错误类型 | 硬件异常 | 可能原因 |
|---------|---------|---------|
| HardFault | HFSR = 0x40000000 | 访问非法地址 / 未对齐访问 |
| MemManageFault | MMFAR = 非法地址 | 访问受保护的内存区域 |
| BusFault | BFAR = 非法地址 | 访问未映射的外设地址 |

**HardFault 调试步骤**：
1. 查看 HFSR 确定异常类型
2. 查看 BFAR/MFAR 获取故障地址
3. 反汇编查看故障地址对应的指令
4. 检查指针是否为 NULL、数组是否越界

## 性能优化诊断

### CPU 占用过高

**诊断方法**：
```c
// 使用 DWT 周期计数器
CoreDebug->DEMCR |= CoreDebug_DEMCR_TRCENA_Msk;
DWT->CYCCNT = 0;

uint32_t start = DWT->CYCCNT;
// ... 执行代码 ...
uint32_t cycles = DWT->CYCCNT - start;
```

**优化方向**：
- 减少浮点运算
- 使用查表法替代复杂计算
- 启用编译器优化（-O2/-O3）

### RAM 使用过高

**诊断方法**：
- 查看 Keil/iar/ld 生成的 map 文件
- 搜索 `.bss`（未初始化全局变量）和 `.data`（已初始化全局变量）段大小

**优化方向**：
- 减小全局数组
- 使用 const 将常量放入 Flash
- 启用局部变量复用优化

### 代码执行时间测量

```c
// 使用定时器测量
uint32_t start_time = TIMx->CNT;
// ... 执行代码 ...
uint32_t elapsed = TIMx->CNT - start_time;
```

## 调试技巧

### 断点使用

| 断点类型 | 用途 | 注意事项 |
|---------|------|---------|
| 硬件断点 | 指令级别 | 数量有限（通常 6-8 个） |
| 软件断点 | C 源码级别 | Flash 中的代码有限制 |
| 数据断点 | 监视变量/内存 | 仅 RAM 区域有效 |

### 变量监视

**最佳实践**：
1. 使用窗口监视关键变量
2. 设置格式（十进制/十六进制）
3. 添加条件监视（如 `count > 100` 时触发）

### 内存查看

**技巧**：
- 查看 RAM：验证栈指针、全局变量值
- 查看 Flash：验证代码是否正确烧录
- 查看外设寄存器：验证配置是否生效

## 总结

故障排查的核心思路：
1. **先验证再修改**：使用工具（示波器/逻辑分析仪）确认问题
2. **按症状分类**：通信/外设/中断/启动/存储
3. **分层检查**：硬件 → 引脚配置 → 外设初始化 → 应用逻辑
4. **记录问题**：在 `编辑清单.md` 中记录故障现象和解决方案
