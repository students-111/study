# IMU / 陀螺仪模块开发检查清单

> 本文件在 RESEARCH 或 EXECUTE 阶段涉及 MPU6050、ICM20602、BMI088、LSM6DS3 等 IMU 传感器的姿态解算时**必须读取并逐项核对**。

---

## 检查项 1：轴匹配（最高优先级）

互补滤波 / 卡尔曼滤波中，加速度计和陀螺仪的轴必须物理一致。

**判断规则**：`atan2` 用了哪两个加速度计轴，**没被用到的那个轴就是旋转轴**，陀螺仪必须用**同名轴**。

| 加速度计公式 | 倾斜平面 | 旋转轴 | 陀螺仪轴 |
|-------------|---------|--------|---------|
| `atan2(accel_x, accel_z)` | XZ | **Y** | **gyro_y** |
| `atan2(accel_y, accel_z)` | YZ | **X** | **gyro_x** |
| `atan2(accel_x, accel_y)` | XY | **Z** | **gyro_z** |

**常见错误**：变量命名为 `GY` 但实际读取了 `gyro_x` 寄存器，导致轴不匹配。必须追溯到寄存器地址（0x43=gyro_x, 0x45=gyro_y, 0x47=gyro_z）确认。

**验证方法**：手持 PCB 绕预期旋转轴缓慢转动，串口打印陀螺仪原始值，确认有明显变化且符号正确。

---

## 检查项 2：量程选择

量程决定了传感器能测量的最大范围。**宁大勿小，饱和比精度低危害大得多。**

### 陀螺仪量程

| 应用场景 | 推荐量程 | 灵敏度 |
|---------|---------|--------|
| 平衡车 / 无人机 / 机器人 | **±2000°/s** | 16.4 LSB/°/s |
| 云台 / 手持稳定 | ±500°/s 或 ±1000°/s | 65.5 / 32.8 |
| 缓慢运动追踪 | ±250°/s | 131 |

**平衡车为什么必须用 ±2000°/s**：快速倾倒纠偏时角速度可达 500~1000°/s，低量程直接饱和截断，互补滤波积分跟不上真实角度，一旦快速倾斜就永远算不回来。

### 加速度计量程

| 应用场景 | 推荐量程 | 灵敏度 |
|---------|---------|--------|
| 有电机振动的场合（平衡车、无人机） | **±16g** | 2048 LSB/g |
| 轻微运动（手环、计步） | ±2g 或 ±4g | 16384 / 8192 |

### 量程-灵敏度绑定关系

**改量程必改灵敏度，全局搜索所有引用点。** 常见需要联动修改的位置：
- 互补滤波中的除数（如 `/ 16384.0f` → `/ 2048.0f`）
- 校准函数中的 1g 基准值（Z 轴减去重力）
- 单位转换函数中的 scale 常量
- 头文件中的默认配置结构体

## 检查项 3：DLPF（数字低通滤波器）

DLPF 在芯片内部对原始数据做硬件平滑。

**核心原则**：滤波器引入的延迟不应超过控制周期的 1/5。

| DLPF 值 | 陀螺仪带宽 | 信号延迟 | 适用场景 |
|---------|-----------|---------|---------|
| 0 | 256Hz | **0ms** | 平衡车、无人机（需要零延迟） |
| 1 | 188Hz | 1.9ms | 快速响应 + 轻度滤波 |
| 2 | 98Hz | 2.8ms | 中等响应 |
| 3 | 42Hz | **4.8ms** | 缓慢运动追踪 |
| 6 | 5Hz | **18.6ms** | 极慢运动 |

**平衡车为什么必须用 DLPF=0**：控制周期 5~10ms，DLPF=3 的 4.8ms 延迟意味着 PID 拿到的数据已过时将近一个控制周期。互补滤波本身已能滤除高频噪声（99% 陀螺仪权重），不需要硬件再滤一遍。

**注意**：DLPF 设置还影响采样率分频基准：
- DLPF=0 时，陀螺仪输出率 = 8kHz，`SampleRate = 8kHz / (1 + SMPLRT_DIV)`
- DLPF≠0 时，陀螺仪输出率 = 1kHz，`SampleRate = 1kHz / (1 + SMPLRT_DIV)`

## 检查项 4：互补滤波系数

```c
Angle = Alpha * AngleAcc + (1 - Alpha) * AngleGyro;
```

Alpha 越小 → 越信任陀螺仪（短期精确，长期漂移）
Alpha 越大 → 越信任加速度计（长期稳定，短期噪声大）

**时间常数**：`τ ≈ dt / Alpha`

| 控制周期 dt | Alpha | 时间常数 τ | 效果 |
|------------|-------|-----------|------|
| 10ms | 0.02 | 500ms | 平衡车典型值 |
| 5ms | 0.01 | 500ms | 等效（周期减半，系数也减半） |

**验证方法**：静止时角度值应稳定在 ±0.5° 以内；快速倾斜 90° 后角度应在 0.5 秒内跟上。

---

## 检查项 5：I2C 通信方式

| 方式 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 软件 I2C（GPIO 位操作） | 时序可控，无总线锁死 | 占用 CPU | **平衡车首选** |
| 硬件 I2C 外设 | CPU 占用低 | CH32V/STM32 硬件 I2C 有已知锁死问题 | 需加超时和总线恢复 |

**中断中做 I2C 读取时**：确保 I2C 操作不会被更高优先级中断打断导致总线状态错乱。

---

## 快速排查流程

当 IMU 姿态解算出现问题时，按以下顺序排查：

```
1. 轴匹配  → atan2 的轴和 gyro 的轴是否对应？（最常见错误）
2. 量程    → 是否饱和？串口打印原始值，快速转动时是否卡在 ±32768？
3. 灵敏度  → 除数是否与量程匹配？改了量程是否全局更新了灵敏度？
4. DLPF    → 延迟是否超过控制周期的 1/5？
5. 滤波系数 → Alpha 和 dt 是否匹配？时间常数是否合理？
6. I2C     → 读取是否成功？是否有丢帧？
7. 符号    → 陀螺仪积分方向是否与加速度计角度方向一致？
```
